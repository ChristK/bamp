## 0. Dependency order (rationale)

```
Phase 1  feature pieces (parallel-safe, ordinary R):   coh-rho → coh-cohdev → coh-Sgt2 → mc-cohcoupling → mc-order
Phase 2  sparse  (large staged R effort; STAGE A overlaps cport STAGE A)
Phase 3  cport   (large staged C effort; consumes the frozen analytic-assembler + RNG-order spec)
```

- **Feature pieces before sparse/C.** Every feature changes the *block layout*, the *constraint set A*, or the *RNG-draw order* (new `rho` MH step, new `dpsi`/`d` blocks, `Omega_psi` Wishart, cause reorder). Both `sparse` and `cport` hard-code the block-assembly and the per-sweep RNG sequence. Landing them first means `sparse`/`cport` are written **once** against the final layout instead of being re-derived after each feature. So: **freeze the R-prototype math, then port.**
- **Within Phase 1, `coh-rho → coh-cohdev → coh-Sgt2`.** `coh-rho` only adds a sampler step (no layout change) — smallest blast radius, do it first and lock the constrained-determinant pattern. `coh-cohdev` adds one disjoint block + one constraint row (small). `coh-Sgt2` is the big coherent rewrite (S deviation blocks, J new constraint rows, layout generalization); it must absorb the `rho` sampler (now per-stratum-shared) and the `dpsi` block, so it goes last among the coherent three. `mc-*` are independent of the coherent three and of each other (different file), so they can proceed in parallel; `mc-cohcoupling` before `mc-order` only because `mc-order` must canonicalise `cor_omega_psi` if cohort coupling exists.
- **`sparse` before `cport`.** Both share **STAGE A** = "stop forming dense X; build Q from an analytic block assembler." Do STAGE A *once*, as the shared correctness oracle. `sparse` then takes the Matrix/CHOLMOD route; `cport` takes the native-C route. `cport`'s `dependsOn:[assemble, rng-order, wishart, constraint-rank]` are all satisfied only after the features + STAGE A are frozen.

## 8. Top cross-cutting correctness traps (watch across all pieces)

1. **Constrained vs unconstrained determinant / df bookkeeping.** Every prior-precision and every Gamma/Wishart df must count *free* dimensions after constraints: ρ uses `logdet(Z'Td Z)` not `log(1-ρ²)`; λ_dc uses K−1 not K; λ_d uses (S−1)·J not S·J; Ω_psi uses K−ord_c not K; κ_theta shares one precision across blocks so `(I-ord_a)` not `(I-ord_a)/2`. A stray off-by-one here biases the precision silently and passes casual tests.
2. **RNG-order / draw-count invariance.** `sparse` and `cport` are bit-reproducible **only** if every sweep consumes exactly `rnorm(P)` for the block draw plus the same gammas/Wishart in the same cell iteration order (coherent `j,i,s`; multicause `j,i,c`). Adding a feature (ρ MH, dpsi gamma, Ω_psi Wishart) **inserts draws** — re-freeze the canonical sequence after Phase 1, before porting. The Wishart upper-triangle fill order is the single most fragile point.
3. **R indexing footguns.** `2:(J-1)` **reverses** to `c(2,1)` at J=2 (guard with `if(J>=3)`); `s$psi[d,,]` / `array[d,,]` **drops** to a vector at Cm=1 (use `matrix(...,nrow,ncol)`, never `t(vec)`); `apply(arr,1,f)` returns a vector not a matrix at the degenerate margin (use `vapply`). These bite at exactly the common 2-cause / J=2 cases.
4. **Closure-binding lag.** `Td` (and any precision matrix read by a `build_prec` closure) must be **reassigned at loop scope** after its hyperparameter changes (ρ step) and **before** the next `build_prec`/`Prec`, or the precision lags one iteration — a slow-mixing bug invisible to a single-step check.
5. **Identifiability is Bayesian (prior-induced), not constraint-induced.** Full row-rank of A guarantees only that `AW` is nonsingular for the Kriging solve — it does **not** identify the APC linear-trend null direction; the intrinsic-RW / proper-AR1 priors do. Every "now identified" claim (coh-Sgt2 `a_s` vs `r_s`, coh-cohdev δ vs dpsi, mc shares) must be stated as *prior*-identified and the weakly-identified quantities reported as their likelihood-identified **sums** (full antisymmetric contribution, full fitted log-odds), not as separately-interpretable blocks. High-ρ regimes erode the prior curvature → mixing pathologies; warn.
6. **Backward compatibility via NULL/`isTRUE` guards.** Old fitted objects lack `samples$rho`, `md$use_dpsi`, `data$order`, `Omega_psi`. Every predict path must fall back (`isTRUE`, `if(!is.null(...))`) so old objects still predict; new fields must be written **every keep** (zeros when a feature is off) so the per-draw vectors have length D and the NULL-fallback fires only for genuinely old objects.
7. **Projection must mean-revert, never free-drift, and must re-center to the fitting constraint.** All deviation/cross-cause forecasts (δ, dpsi, d_{s,j}, coupled Ψ) use AR1 `.cj_predict_ar`/`.mvrw_predict`, never `.cj_predict_rw`; and any per-period sum-to-zero enforced in the fit must be **re-imposed at every projected period** (re-center over strata/causes) or the independently-simulated paths re-introduce a spurious shared trend that leaks into the total. Watch the Sørbye-Rue scale inflation (`s_c` up to hundreds) on long-K RW2.
